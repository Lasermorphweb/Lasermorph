🧪 懒加载测试验证工具

验证视频和图片懒加载是否正确工作

📋 测试说明

测试步骤：
  1. 打开 Chrome DevTools (按 F12 或右键→检查)
  2. 切换到 Network (网络) 面板
  3. 勾选 "Disable cache" (禁用缓存)
  4. 刷新页面并观察加载的资源
  5. 检查是否有视频在页面加载时立即下载

✅ 预期结果

优化生效的标志：

  • 页面加载时 - Network 面板中没有 .mp4 文件
  • 滚动到视频位置时 - 才看到 .mp4 文件开始下载
  • Console 面板 - 看到 "✅ 加载视频" 的日志
  • HTML 源码 - video 标签使用 data-src 而不是 src

优化未生效的标志：

  • ❌ 页面加载时就有 mp4 文件下载
  • ❌ HTML 中仍有 src 属性
  • ❌ 视频有 autoplay 属性

🔧 测试工具

打开浏览器控制台 (F12 → Console)，执行以下命令：

1. 检查视频加载状态

window.checkVideoStatus()

返回：{ loaded: X, pending: Y }

loaded = 已加载的视频数，pending = 待加载的视频数

2. 手动加载所有视频（测试用）

window.loadAllVideos()

这会强制加载所有懒加载的视频

3. 检查HTML结构

// 检查是否使用 data-src
document.querySelectorAll('video source[data-src]').length

// 检查是否有错误的 src
document.querySelectorAll('video source[src]').length

🚨 已修复的问题

✅ 已修复：

  1. 视频预加载问题 - 所有 <source src> 改为 <source data-src>
  2. 自动播放问题 - 移除了所有 autoplay 属性
  3. 懒加载标记 - 添加了 data-lazy="true" 标记
  4. 预加载控制 - 设置了 preload="none"

📝 技术细节

修改前：

<video autoplay loop muted playsinline>
    <source src="video.mp4" type="video/mp4">
</video>

修改后：

<video data-lazy="true" preload="none" loop muted playsinline>
    <source data-src="video.mp4" type="video/mp4">
</video>
关键差异：
  • src → data-src (浏览器不会自动加载)
  • 移除 autoplay (避免强制加载)
  • 添加 data-lazy="true" (标记为懒加载)
  • JavaScript 在视频进入视口时才设置 src 并加载
