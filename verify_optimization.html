🔍 网站性能优化验证报告

❌ 发现的关键问题

问题 1: 懒加载脚本无法阻止视频预加载

原因：

  • HTML 中已经写死了 <source src="...">
  • 浏览器在 DOMContentLoaded 之前就开始加载视频
  • 脚本执行时移除 src 已经太晚了

影响：

  • Design Tool 页面会在加载时尝试加载 50+ 个视频
  • 即使视频不可见，也会消耗带宽

问题 2: HTML 中的 autoplay 属性强制加载

原因：

  • 所有视频都有 autoplay 属性
  • autoplay 会强制浏览器立即加载视频
  • 即使设置了 preload="none" 也无效

问题 3: 图片尺寸过大

发现 38 个超过 5MB 的图片，总计 301.42 MB

最大的文件：

  • 19.88 MB - Rectangle 114.png
  • 18.34 MB - fm_Rectangle 112.png
  • 18.16 MB - Rectangle 113.png

✅ 正确的优化方案

方案 1: 使用 data-src 替代 src（必须）

修改前：

<video autoplay loop muted playsinline>
    <source src="../../resources/video.mp4" type="video/mp4">
</video>

修改后：

<video loop muted playsinline data-lazy="true">
    <source data-src="../../resources/video.mp4" type="video/mp4">
</video>

关键点：

  • ✅ 移除 autoplay 属性
  • ✅ 使用 data-src 而不是 src
  • ✅ 添加 data-lazy="true" 标记
  • ✅ 由 JavaScript 控制加载时机

方案 2: 优化后的懒加载脚本

document.addEventListener('DOMContentLoaded', function() {
    // 查找所有需要懒加载的视频
    const lazyVideos = document.querySelectorAll('video[data-lazy]');
    
    const videoObserver = new IntersectionObserver((entries) => {
        entries.forEach(entry => {
            if (entry.isIntersecting) {
                const video = entry.target;
                const source = video.querySelector('source[data-src]');
                
                if (source && source.dataset.src) {
                    // 设置 src 并加载
                    source.src = source.dataset.src;
                    video.load();
                    video.play().catch(e => console.log('Autoplay blocked'));
                    video.removeAttribute('data-lazy');
                }
            }
        });
    }, { rootMargin: '50px' });
    
    lazyVideos.forEach(video => videoObserver.observe(video));
});

  2. 移除所有 autoplay 属性

  4. 测试验证 - 打开 DevTools Network 面板确认

🔬 如何验证优化是否生效

打开 Chrome DevTools (F12) → Network 面板：

  1. 刷新页面
  2. 观察是否有视频在页面加载时就开始下载
  3. 如果看到视频立即下载 → 优化未生效 ❌
  4. 如果滚动后才看到视频下载 → 优化生效 ✅
